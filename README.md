# [Hillel C# Pro | ДЗ 2. Pull request](https://lms.ithillel.ua/groups/645d0f9f0c133308618837f5/homeworks/64fb1f1a65b85a02530fde62)

*lets follow best coding practices - pull requests*

Будемо тренуватись працювати в команді, робити в одному репозиторії таски разом, створювати та ревьювити пул ріквести.

Ось детальний покроковий опис завдання:

- [ ] Створюємо проект (це робить хтось один з команди)
- [ ] Додаємо до репозиторію Github (це робить хтось один з команди)
- [ ] Клонуємо собі проект з пунктів 1,2 (всі інші в команді, окрім того хто його створив)
- [ ] Створюємо бранчу (кожен в команді)
- [ ] Вносимо будь-які зміни. (кожен в команді)
- [ ] Створюємо Pull request у Github (кожен в команді)
- [ ] Кидаємо в кожному зі своєї команди цей пул ріквест на рев'ю (кожен в команді)
- [ ] Рев'ювимо пул ріквести інших членів команди.(кожен в команді) (треба додати якісь коментарі, і в кінці кінців аппрувнути пул ріквест)
- [ ] Далі коли ваш особистий пул ріквест апрувнуло хоча б 2 людини - ви можете його мержити в master (звичайно пофіксивши мерж конфлікти якщо такі будуть).
- [ ] Кидаєте мені лінку на репозиторій
- [ ] Скріни з вашого GitExtention з вашою бранчою і змінами
- [ ] Лінку на пул ріквест
